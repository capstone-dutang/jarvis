# 5주차 주간보고서

## 금주예정작업:

이번 주에는 JARVIS 백엔드 MVP 구현을 시작하는 것을 목표로 하였다. FastAPI 기반 서버를 구성하고, PostgreSQL 데이터베이스와 연결하며, SQLAlchemy 모델과 Alembic 마이그레이션을 통해 기본 테이블을 생성하는 작업을 진행할 예정이었다. 또한 Docker Compose를 이용해 팀원들이 동일한 로컬 환경에서 서버와 데이터베이스를 실행할 수 있도록 구성하는 것도 중요한 예정 작업이었다.

기능적으로는 워크스페이스 생성, 대화 episode 저장, 기본 검색 API, 헬스 체크 endpoint를 우선 구현 대상으로 정하였다. 이 단계에서는 고급 검색 품질보다 서버가 실제로 실행되고 데이터를 저장할 수 있는 기반을 만드는 것을 목표로 하였다.

## 금주 수행결과 및 문제점 해결방법:

금주에는 FastAPI 서버의 기본 구조와 PostgreSQL 연결 구성을 구현하였다. SQLAlchemy 기반 모델 구조를 만들고 Alembic 마이그레이션을 통해 초기 스키마를 관리할 수 있도록 하였다. Docker Compose 설정을 추가하여 데이터베이스와 서버를 함께 실행할 수 있는 개발 환경을 구성하였다. 또한 `/health` endpoint를 통해 서버 상태를 확인할 수 있도록 하여 이후 배포와 테스트의 기준점을 마련하였다.

문제점으로는 초기 데이터 모델이 구현 과정에서 계속 변경된다는 점이 있었다. Episode, Entity, KnowledgeFact의 관계를 단순하게 잡으면 구현은 쉬웠지만, 이후 회상 검색에서 근거와 관계를 충분히 표현하기 어려웠다. 이를 해결하기 위해 초기 단계부터 대화 원문과 구조화된 사실을 분리하고, 추후 검색 품질 개선을 위해 fragment 저장소를 추가할 수 있는 확장 가능한 구조로 정리하였다.

## 지도교수 면담 및 토론결과:

지도교수 면담에서는 MVP 구현에서 가장 먼저 보여줄 수 있는 결과가 무엇인지 논의하였다. 교수님은 “설계가 복잡하더라도 사용자가 볼 수 있는 동작 흐름이 있어야 한다”고 조언하였다. 이에 따라 저장 API와 회상 API를 우선 완성하고, 이후 웹 UI와 고급 검색을 붙이는 순서로 개발 우선순위를 조정하였다.

또한 DB 스키마 변경이 잦을 수 있으므로 마이그레이션 기록을 남겨두는 것이 중요하다는 의견이 있었다. 이 피드백을 반영하여 Alembic 기반으로 스키마 변화를 관리하기로 하였다.
