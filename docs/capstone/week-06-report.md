# 6주차 주간보고서

## 금주예정작업:

이번 주에는 JARVIS의 워크스페이스 구조와 맥락 저장 흐름을 구체화하는 것을 목표로 하였다. 여러 프로젝트의 대화가 하나의 저장소에 섞이면 회상 결과가 부정확해질 수 있으므로, 프로젝트 단위로 기억을 분리하고 필요할 때 해당 워크스페이스의 맥락만 불러오는 구조를 구현할 예정이었다. 또한 대화에서 생성된 사실을 즉시 확정 정보로 처리할지, 검토 가능한 형태로 관리할지에 대한 정책도 정리하고자 하였다.

이와 함께 사용자, 워크스페이스, episode, fact 간의 관계를 실제 API 흐름으로 연결하는 작업을 진행하기로 하였다. 최종적으로는 특정 워크스페이스에 대화 내용을 저장하고, 이후 같은 워크스페이스에서 회상 검색을 수행할 수 있는 흐름을 만드는 것을 목표로 하였다.

## 금주 수행결과 및 문제점 해결방법:

금주에는 워크스페이스 중심의 기억 관리 구조를 구현하였다. 프로젝트별 기억 공간을 분리하기 위해 Workspace 모델과 관련 API를 정리하고, Episode와 KnowledgeFact가 특정 워크스페이스에 속하도록 구성하였다. 이를 통해 한 프로젝트의 대화 내용이 다른 프로젝트의 회상 결과에 섞이지 않도록 기본적인 격리 구조를 만들었다.

문제점으로는 대화에서 추출된 내용의 신뢰도를 어떻게 다룰 것인지가 있었다. AI가 요약하거나 구조화한 내용은 유용하지만, 항상 정확하다고 보기는 어렵다. 이를 해결하기 위해 원본 대화 또는 source_quote를 함께 저장하여 나중에 근거를 확인할 수 있도록 설계하였다. 또한 모든 사실을 덮어쓰기보다 시간 순서대로 누적하고, 변경된 판단도 기록으로 남기는 방향을 채택하였다. 이 방식은 JARVIS를 단순한 최신 상태 저장소가 아니라 작업 변화 과정을 담는 일기형 기억 시스템으로 발전시키는 기반이 되었다.

## 지도교수 면담 및 토론결과:

지도교수와의 면담에서는 프로젝트별 기억 분리가 실제 사용성에 중요하다는 의견을 받았다. 교수님은 하나의 저장소에 모든 대화가 들어갈 경우 검색 결과가 많아질수록 오히려 혼란이 생길 수 있다고 지적하였다. 이에 따라 워크스페이스 개념을 최종보고서에서 중요한 설계 요소로 설명하기로 하였다.

또한 AI가 생성한 요약을 그대로 믿는 문제에 대해서도 논의하였다. 교수님은 평가자가 신뢰성 문제를 질문할 수 있으므로, 저장된 사실이 어떤 대화에서 나왔는지 근거를 함께 제시하는 구조가 필요하다고 조언하였다. 이 피드백을 반영하여 source_quote와 episode 기반 근거 추적을 설계에 포함하였다.
