# 12주차 주간보고서

## 금주예정작업:

이번 주에는 JARVIS의 기억 저장 방식을 “AI가 쓰는 일기” 모델로 정리하고, 이를 확인할 수 있는 웹 UI를 구현하는 것을 목표로 하였다. 기존에는 대화 원문과 구조화된 fact 저장에 초점이 있었지만, 실제 사용자는 날짜별로 어떤 작업을 했는지, 어떤 주제를 다뤘는지, 무엇이 결정되었는지를 한눈에 확인할 필요가 있다. 따라서 episode를 단순 로그가 아니라 diary entry, summary, human summary, turns, facts를 포함하는 기록 단위로 재정의하고자 하였다.

또한 브라우저에서 JARVIS에 저장된 기억을 확인할 수 있도록 단일 페이지 웹 UI를 구현할 예정이었다. 날짜별 일기, 주제 목록, 검색, 엔티티 정보를 확인하는 흐름을 MVP 수준으로 만드는 것이 목표였다.

## 금주 수행결과 및 문제점 해결방법:

금주에는 `jarvis_log_diary`를 중심으로 한 일기 저장 모델을 정리하였다. raw transcript를 통째로 업로드하는 방식은 토큰 부담이 크고, Claude 앱처럼 로컬 파일 접근이 없는 환경에서는 사용할 수 없다는 한계가 있었다. 이를 해결하기 위해 AI가 자기 컨텍스트 안에서 대화 내용을 재구성하여 turns, summary, diary_entry, human_summary, keywords, entities, facts, relations를 한 번에 기록하는 구조를 채택하였다.

웹 UI MVP도 구현하였다. 단일 파일 SPA 형태로 구성하여 FastAPI에서 정적 파일로 서빙하고, 사용자가 날짜별 episode와 주요 주제를 확인할 수 있도록 하였다. 문제점은 UI에서 너무 많은 정보를 한 번에 보여주면 오히려 사용성이 떨어진다는 점이었다. 이를 해결하기 위해 홈 화면에서는 최근 작업과 주요 주제를 먼저 보여주고, 필요한 경우 episode 상세나 엔티티 정보를 펼쳐보는 방식으로 정리하였다.

## 지도교수 면담 및 토론결과:

지도교수 면담에서는 일기 모델이 프로젝트 설명에 도움이 된다는 의견을 받았다. 교수님은 “지식 그래프”나 “벡터 검색” 같은 기술 용어만으로는 서비스의 목적이 추상적으로 보일 수 있으므로, “AI가 작업 일기를 쓰고 다음 세션에서 회상한다”는 표현이 평가자에게 더 직관적일 것이라고 조언하였다.

또한 웹 UI는 모든 기능을 완성하기보다 시연에서 핵심 흐름을 보여주는 데 집중하라는 피드백을 받았다. 이에 따라 최종 UI는 날짜별 기록 확인, 검색, 주요 주제 확인에 초점을 맞추기로 하였다.
