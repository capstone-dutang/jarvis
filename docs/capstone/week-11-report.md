# 11주차 주간보고서

## 금주예정작업:

이번 주에는 JARVIS의 회상 기능을 단순 fact 검색에서 narrative context 검색까지 확장하는 것을 목표로 하였다. 사용자는 과거에 어떤 결정을 했는지만 알고 싶은 것이 아니라, 왜 그런 결정을 했는지, 어떤 대안이 있었는지, 어떤 문제를 해결했는지를 함께 알고 싶어 한다. 따라서 구조화된 KnowledgeFact 검색만으로 부족한 경우 대화 본문 일부를 다시 찾아볼 수 있는 도구가 필요하였다.

구체적으로는 주제 탐색, passage 검색, episode 발췌, 관계 따라가기 기능을 추가하여 회상 흐름을 단계별로 구성할 예정이었다. 사용자가 넓은 질문을 하면 먼저 관련 주제 지형을 보여주고, 특정 사실을 찾은 뒤에는 해당 episode의 대화 맥락으로 들어갈 수 있도록 설계하였다.

## 금주 수행결과 및 문제점 해결방법:

금주에는 `jarvis_explore_topic`, `jarvis_search_passages`, `jarvis_get_episode_excerpt`, `jarvis_follow_relation` 기능을 추가하였다. `explore_topic`은 특정 주제 주변에 어떤 엔티티와 관계가 있는지 확인하는 용도로 사용된다. `search_passages`는 구조화된 fact가 부족할 때 자연어 passage를 검색하는 기능이고, `get_episode_excerpt`는 특정 episode 안에서 질문과 관련된 부분을 발췌하는 기능이다. `follow_relation`은 특정 엔티티와 연결된 주변 엔티티를 따라가며 지식 그래프를 탐색할 수 있도록 한다.

문제점은 도구가 많아질수록 AI가 어떤 도구를 먼저 써야 하는지 혼란스러워질 수 있다는 점이었다. 이를 해결하기 위해 각 MCP 도구 설명에 사용 조건과 사용하지 말아야 할 조건을 명확히 작성하였다. 또한 기본 회상은 `jarvis_recall_memory`로 시작하고, 결과가 좁거나 맥락이 부족할 때 보조 도구를 사용하는 2단계 회상 패턴을 정리하였다. 이를 통해 JARVIS의 회상 흐름이 더 자연스럽고 설명 가능해졌다.

## 지도교수 면담 및 토론결과:

지도교수와의 토론에서는 “회상”이라는 기능을 평가자가 이해하기 쉽게 시연해야 한다는 의견이 있었다. 교수님은 단순히 검색 결과를 보여주는 것보다, 사용자가 질문을 던지고 AI가 과거 결정과 근거를 찾아오는 흐름을 보여주는 것이 설득력 있다고 조언하였다.

또한 여러 MCP 도구가 존재하더라도 최종 발표에서는 핵심 도구 몇 개만 중심으로 설명하는 것이 좋다는 피드백을 받았다. 이에 따라 발표에서는 초기화, 일기 저장, 회상 검색, episode 발췌를 중심으로 시연 흐름을 구성하기로 하였다.
