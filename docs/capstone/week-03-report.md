# 3주차 주간보고서

## 금주예정작업:

이번 주에는 2주차에 정의한 문제를 바탕으로 JARVIS의 핵심 기능 범위와 시스템 개념을 구체화하는 것을 목표로 하였다. 단순히 대화 내용을 저장하는 수준을 넘어서, AI가 다시 사용할 수 있는 기억 구조가 무엇인지 정의하고자 하였다. 특히 워크스페이스, 세션, 대화, 사실, 엔티티, 관계와 같은 개념을 분리하여 데이터 모델의 초기 방향을 잡는 것이 주요 예정 작업이었다.

또한 캡스톤 프로젝트 범위 안에서 구현할 기능과 향후 확장 기능을 구분하기로 하였다. MVP에서는 AI 대화 저장, 워크스페이스 분리, 자연어 회상, 기본 웹 UI, MCP 연동을 중심으로 구현하고, 팀 권한 관리나 고급 그래프 시각화 같은 기능은 향후 개선 항목으로 분류하는 것을 계획하였다.

## 금주 수행결과 및 문제점 해결방법:

금주에는 JARVIS의 핵심 비전을 “AI가 사용하는 클라우드 기억 저장소”로 정리하였다. 프로젝트의 주요 사용 흐름은 사용자가 AI와 대화하면서 발생한 핵심 내용을 JARVIS에 저장하고, 이후 다른 세션이나 다른 AI 클라이언트에서 이를 회상하는 방식으로 정의하였다. 이를 통해 JARVIS가 단순 노트 앱과 다른 점은 사람이 직접 찾기 위한 저장소가 아니라 AI가 작업 맥락을 다시 불러오기 위한 저장소라는 점을 명확히 하였다.

기능 범위를 정리하는 과정에서 처음에는 todo, 일정, 알림, 파일 관리 등 여러 개인 비서 기능이 포함될 가능성이 있었으나, 캡스톤 기간 안에서 모두 구현하기 어렵다는 문제가 있었다. 이를 해결하기 위해 핵심 기능을 기억 저장과 회상으로 제한하고, 나머지는 향후 확장 가능성으로 남겼다. 또한 “프로젝트별 기억 공간”이라는 개념을 도입하여 여러 프로젝트의 맥락이 섞이지 않도록 워크스페이스 단위를 설계하였다.

## 지도교수 면담 및 토론결과:

지도교수 면담에서는 프로젝트의 차별성을 어떻게 설명할 것인지에 대해 논의하였다. 교수님은 기존 문서 관리 시스템, 챗봇, 메모 앱과의 차이를 평가자가 쉽게 이해할 수 있도록 정리해야 한다고 조언하였다. 이에 따라 JARVIS의 차별점을 “AI 클라이언트가 직접 기억을 저장하고 회상한다”, “서로 다른 AI 세션 사이에서 맥락이 이어진다”, “단순 로그가 아니라 구조화된 지식으로 저장한다”는 세 가지로 정리하였다.

또한 기능 명세를 너무 추상적으로 작성하면 구현 결과와 연결되기 어렵다는 의견이 있었다. 이에 따라 다음 주부터는 데이터 모델과 API 단위로 구체적인 설계를 진행하기로 하였다.
