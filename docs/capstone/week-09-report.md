# 9주차 주간보고서

## 금주예정작업:

이번 주에는 JARVIS의 핵심 기능인 회상 검색을 고도화하는 것을 목표로 하였다. 단순 키워드 검색만으로는 과거 대화의 의미를 충분히 찾기 어렵고, 의미 검색만으로는 정확한 기술명이나 고유명사를 놓칠 수 있다. 따라서 pgvector 기반 의미 검색, PGroonga 기반 전문 검색, EntityRelation 기반 그래프 탐색을 결합하는 하이브리드 검색 구조를 구현할 예정이었다.

또한 검색 결과를 AI가 바로 사용할 수 있도록 관련 사실, 근거, 관련 엔티티를 함께 반환하는 구조를 설계하고자 하였다. 최종 목표는 사용자가 “그때 왜 PostgreSQL을 선택했지?”와 같이 질문했을 때, 단순히 PostgreSQL이 언급된 문장을 찾는 것이 아니라 선택 이유와 관련 맥락을 함께 회상하는 것이다.

## 금주 수행결과 및 문제점 해결방법:

금주에는 pgvector, PGroonga, 그래프 탐색을 결합한 하이브리드 검색 구조를 구현하였다. KnowledgeFact와 Fragment에 저장된 내용을 대상으로 의미 검색을 수행하고, 키워드 기반 전문 검색 결과와 결합하여 더 넓은 후보를 얻도록 하였다. 또한 EntityRelation을 통해 특정 엔티티와 연결된 다른 엔티티를 탐색할 수 있도록 하여, 사용자의 질문이 특정 개념에 걸려 있을 때 주변 맥락까지 함께 회수할 수 있게 하였다.

문제점은 검색 결과가 많아질수록 AI에게 전달할 맥락이 과도하게 길어질 수 있다는 점이었다. 이를 해결하기 위해 검색 결과를 점수화하고, 중복되는 내용은 줄이며, 관련성이 높은 결과를 우선 반환하는 방향으로 정리하였다. 또한 추후 MMR 방식의 다양성 재정렬을 적용할 수 있도록 검색 결과 구조를 정돈하였다. 이 단계에서 JARVIS는 단순 저장 API를 넘어 실제 회상 시스템으로 동작할 수 있는 기반을 확보하였다.

## 지도교수 면담 및 토론결과:

지도교수 면담에서는 검색 방식을 여러 개 결합하는 이유를 명확히 설명해야 한다는 피드백을 받았다. 교수님은 “왜 벡터 검색만으로 충분하지 않은가”를 평가자가 물을 수 있다고 지적하였다. 이에 대해 의미 검색은 표현이 달라도 비슷한 내용을 찾는 데 강하지만, 프로젝트명이나 기술명 같은 정확한 키워드는 전문 검색이 더 유리하다고 정리하였다. 또한 그래프 탐색은 특정 개념의 주변 관계를 찾는 데 필요하다고 설명하였다.

이 논의를 바탕으로 최종보고서의 검색 구조 장에는 세 가지 검색 방식의 역할을 분리하여 서술하기로 하였다.
