## 진행할 4단계 (남은 일)

### 1. MCP 도구 설명서 갱신

**목적**: 다른 세션의 AI(사용자의 코딩 에이전트)가 자비스에 접속했을 때, "오늘 대화 자비스에 올려"라는 사용자 명령을 받고 어떤 도구를 어떤 순서로 호출해야 하는지 스스로 판단할 수 있게 함.

**대상 파일**: `jarvis/src/jarvis/mcp_adapter.py` (또는 그 안의 도구 정의/설명) — 정확한 위치는 작업 진입 시 확인.

**갱신 내용**:
- 기존 `store_memory` 단독 호출 가이드 → 새 통합 흐름 가이드
- 권장 호출: `/ingest-and-index` 한 번
- 회상 시: `/recall` (entity-anchored hybrid)
- 사용자가 "오늘 대화 올려"라고 했을 때 AI가 따를 정확한 절차

**검수 포인트 (사용자 필수)**: 작업 끝나면 한 번 멈춰서 사용자 검수 받은 다음 2번 진행.
- 이유: MCP 도구 설명은 비전을 다른 AI에게 전달하는 매뉴얼. 사용자 의도와 일치해야 함.

---

### 2. MCP 실사용 검증

**목적**: 지금까지의 검증은 다 로컬 HTTP API 직접 호출. 진짜 시나리오("다른 세션 AI가 MCP로 자비스 접속 → 트랜스크립트 올림 → 또 다른 세션에서 회상")가 작동하는지 확인.

**검증 시나리오**:
1. 새 Claude Code 세션을 띄움
2. 자비스 MCP 도구가 보이는지 확인
3. "오늘 대화 자비스에 올려" 명령 → AI가 알아서 `/ingest-and-index` 호출하는지
4. 또 다른 세션에서 "그 때 X 어땠지?" 같은 질문 → AI가 `/recall` 호출 + 답변 가져오는지

**산출**: 작동/미작동 항목 정리, 미작동 시 개선 후 재검증.

---

### 3. 데이터 쫙 올리며 QA

**목적**: 가진 데이터로 자비스를 진짜 채우면서 회상 품질을 raw ↔ 결과물 대조로 검증.

**QA 기준** (단순 "데이터 올라갔다"가 아닌, 회상 품질):
- 사용자가 평소 자비스에 묻고 싶었던 **실제 쿼리** 5~10개 직접 던지기 (제가 만든 시나리오는 편향)
- 자비스 답변(top-N fact + 발췌)을 raw 트랜스크립트와 대조
- "이 답이 충분한가 / 빠진 게 있는가"를 사용자가 평가
- 빠진 게 있으면 원인 진단(분류 누락/색인 부족/회상 알고리즘 한계) → 개선

**개선 사이클**: 발견 → 진단 → 수정 → 다시 올림. 이 단계에서 코드 변경 더 들어갈 수 있음을 인정.

**데이터 범위 결정**: 3번 진입 시점에 결정.
- `~/.claude/projects/F--brain/` 전체 jsonl을 다 올릴지
- 일부만 올릴지
- 다른 워크스페이스(argos-crypto 등)를 따로 둘지

### 4. UI 마무리

**화면 구조** (비전 §4 기준):

**진행 시점**: 1, 2, 3번 다 끝난 뒤.

## 짚을 점 (전 과정 공통)

**(a) QA의 기준은 사용자 시각**
"데이터 올라갔다"가 아닌, "사용자가 진짜 궁금한 쿼리에 자비스가 만족스럽게 답하는가". 사용자가 직접 쿼리 던져서 검증.

**(b) MCP 도구 설명서 검수는 AI 자체 테스트** *(2026-05-13 사용자 정정)*
MCP 도구 docstring·instructions·시그니처는 다른 세션의 AI가 읽는 매뉴얼. 사용자 검수 대상이 아님. 매뉴얼만 보고 AI가 사용자 명령에 맞는 도구를 부르는지 시뮬레이션/실호출 테스트로 검수. 결과만 사용자에게 보고.

**(c) 데이터 범위 — 잠정 default**
- `~/.claude/projects/F--brain/` jsonl: ai-clean-test 워크스페이스에 누적 (자비스 본체 작업 출처)
- `~/.claude/projects/F--argos-crypto/` 등 다른 프로젝트 jsonl: 별도 워크스페이스 (예: ai-argos)
- 1프로젝트 = 1워크스페이스 원칙
- 3번 진입 시 사용자 확인 후 조정 가능

## 자비스 한 줄 정체성 (다시 확인)

> **자비스 = AI(사용자의 코딩 에이전트)가 사용자 대화의 일기를 클라우드에 쓰는 시스템.**

- AI가 일기 작성자 (정제 + 요약 + 키워드 + 분류 + entity/fact 색인 모두 AI)
- 클라우드 = 완전 백업 + 색인 (raw도 자비스 안)
- KG의 supersede/NLI 끔 → 일기는 누적이지 갱신이 아님
- 모순도 정보 (생각이 바뀐 것 자체가 기록)
- 회상 시 entity-anchored hybrid recall로 최소 토큰 맥락 회수
